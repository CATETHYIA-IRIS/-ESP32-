# 灵汐硬件接线核对表

本文用于答辩前、换板前、重新接线前快速核对硬件连接。当前推荐部署方式是：ESP32-S3 中控 + C3 感知子节点 A/B + C3 执行子节点 A/B。集成感知层和集成执行层作为兼容/早期方案一并记录。

## 总原则

- ESP32 GPIO 不耐 5V。所有接入 GPIO/ADC/I2C/UART 的信号必须是 3.3V 电平，5V 模块输出需要分压、隔离或确认模块 DO/AO 已经是 3.3V。
- 传感器模块、继电器模块、电机驱动、舵机、灯带如果使用外部电源，必须与 ESP32 共地。
- 电机、舵机、继电器线圈、灯带不要从 ESP32 的 3.3V 引脚取电。ESP32 只输出控制信号。
- 上电前先断开执行器负载，确认中控和节点在线后，再逐个接入灯带、继电器、电机和舵机。
- C3 通用执行板中 IO9 保留 BOOT，IO18/IO19 保留 USB，IO20/IO21 尽量保留串口；代码中用到 IO2 时，外部电路不要强拉高/低，以免影响启动。
- 节点默认接入中控热点时，中控地址为 `192.168.4.1`；如果中控和节点都接入路由器，则节点网关地址改为中控局域网 IP。

- SD 卡推荐 FAT32，根目录包含 `/record`、`/voice/shorekeeper`、`/voice/cartethyia`、`/roles/shorekeeper`、`/roles/cartethyia`。
- `/roles` 必须在 SD 根目录，不要放进 `/voice/roles`。
- 中控默认热点 `Lingxi-Gateway`，默认密码 `12345678`，正式演示前可在 WebUI 或源码中修改。
- 串口监视器波特率为 `921600`。
- 中控作为系统网关，先确认 WebUI 正常，再给节点上电。

### 执行 A 高风险点

- 继电器为低电平触发：`LOW` 开启，`HIGH` 关闭。接反会导致上电误动作。
- 继电器只负责开关信号，高压侧必须按电工规范接线，答辩演示优先使用低压灯/小负载。
- WS2812 灯带电流按最大白光估算，8 颗灯满白约数百 mA，供电余量要够。

- 电机第一次接线先不要带窗帘负载，确认方向、步数和限位后再装入机构。

## 上电前总检查

- [ ] 中控 SD 卡已插入，目录结构正确。
- [ ] 中控先上电，WebUI 能打开。
- [ ] 节点 WiFi 已配置到正确网络，网关地址正确。
- [ ] 所有外部 5V 电源负极与 ESP32 GND 共地。
- [ ] 传感器输出电平不超过 3.3V。
- [ ] MQ2/MQ5 模拟输出接 ADC 前确认最高电压。
- [ ] 继电器模块确认低电平触发，上电默认关闭。
- [ ] 电机/舵机未带负载完成方向测试。
- [ ] 高压负载不裸露，答辩优先低压演示。
- [ ] 每次只接入一个新模块，确认日志和状态正常后再接下一个。

## 异常速查

| 现象 | 优先检查 |
| --- | --- |
| C3 节点不上电或反复重启 | 5V/3.3V 供电是否稳定，IO2/IO8/IO9 是否被外部电路异常拉动 |
| 节点能开热点但不上线 | WiFi SSID/密码、网关地址、是否连接中控热点或同一路由器 |
| 门磁状态反了 | `CONTACT_OPEN_LEVEL` 语义、模块 DO 电平、磁铁位置 |
| PIR 经常误报 | 输入是否悬空、模块供电是否稳定、PIR 触发保持时间 |
| MQ 数值异常 | 预热时间、AO 电压、阈值、传感器供电 |
| 继电器上电动作 | 触发电平、IN 线接错、供电瞬态、负载侧接线 |
| WS2812 不亮或颜色乱 | 5V 供电、电流不足、共地、数据线方向、灯珠数量 |
| 步进电机只抖不转 | ULN2003 跳线帽、IN1-IN4 线序、外部 5V 电流不足 |
| 舵机抖动/复位 | 舵机电源不足、未共地、机械卡死 |
